Event Streams and Processors

The previous example introduced two important concepts of BeepBeep 3: event streams and processors.

An event stream (also called a trace) is a sequence of elements called events. In BeepBeep 3, an event can be anything: a number, a character string, a tuple of name-value pairs, an XML document, a set, or any other data structure. In the "Hello World" example, the expression "HELLO WORLD" refers to the infinite event stream where each event is the same character string. Here are some examples of event streams:

1, 2, 3, 4, 5, ...

"abc", "def", "abc", ...

(p=2, q=3), (p=0, q=0), ...

A processor is a function taking zero or more event streams as its input, and producing zero or more event streams as its output. As an example, consider the EVERY processor, which returns every n-th event of its input stream. Hence if S is the stream 1, 2, 3, 4, 5, …, the expression:

EVERY 2ND OF (S)

produces the output stream 1, 3, 5, 7, ...

Processors can be piped: the output of one processor can be given as the input of another.
